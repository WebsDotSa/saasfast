الخطوات التالية — مرتّبة بالأهمية

المشروع وصل 95% — ما تبقى هو إعدادات وليس كوداً.
1
🔴 تطبيق Migrations 025, 026, 030-033, 040-044 على Supabase
أهم خطوة — 14 migration لم تُطبَّق. بدونها كل Marketing + Payments + AI APIs ستفشل. في Supabase SQL Editor، شغّل كل ملف بالترتيب.
تحقق أولاً: SELECT table_name FROM information_schema.tables WHERE table_name IN ('discounts', 'store_transactions', 'ai_agents', 'merchant_balances');
2
🔴 أضف DATABASE_URL في .env.local
من Supabase → Settings → Database → Connection string (Direct connection). بدونه lib/db/index.ts يفشل وتنهار كل AI Agent APIs رغم أن الكود جاهز 100%.
3
🔴 أضف ANTHROPIC_API_KEY الحقيقي في .env.local
الحالي: sk-ant-your_anthropic_key_here — لن يعمل. من console.anthropic.com أنشئ مفتاحاً حقيقياً. Marketing AI يعمل بـ Qwen عبر .env.ai، لكن AI Agent يحتاج Anthropic مباشرة.
4
🔴 أصلح model name في ملفين
في lib/db/schema.ts (default value) وفي api/ai/agents/route.ts: غيّر 'claude-sonnet-4' ❌ إلى 'claude-sonnet-4-5' ✓
5
⚠️ انقل .env.ai لـ .env.local
ANTHROPIC_AUTH_TOKEN + ANTHROPIC_BASE_URL + ANTHROPIC_MODEL موجودة في ملف منفصل .env.ai. انقلها أو أضف import له في الكود للتأكد من تحميلها.
6
⚠️ أضف Marketing رابط في Sidebar التاجر
تم بناء 9 صفحات marketing لكن لا يوجد رابط في components/dashboard/sidebar.tsx للتنقل إليها. أضف { title: 'التسويق', href: '/dashboard/marketing', icon: Megaphone }
7
⚠️ اختبر Payments + Marketing بعد تطبيق Migrations
بعد تطبيق الـ migrations: افتح /dashboard/payments وتحقق الرصيد، افتح /dashboard/marketing وتحقق الخصومات، اختبر discount validation API.
8
✅ أضف Onboarding Guard في middleware.ts
المستخدم الجديد الذي لم يختر نشاطه يجب توجيهه لـ /onboarding/business-type. سطر شرط واحد في middleware.ts.
9
🔧 ربط Cloudflare API للنطاقات المخصصة
domains/route.ts يحفظ في DB لكن لا يتصل بـ Cloudflare API فعلياً. تحتاج تفعيل الاتصال الحقيقي باستخدام CLOUDFLARE_API_TOKEN المضبوط.
